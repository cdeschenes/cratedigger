Help & Documentation

Cratedigger scans your Navidrome library against Last.fm to surface albums you're missing and artists you might enjoy.

Getting Started

  1. 1 Go to Run Dashboard and click Run Now on either job. Both scripts connect to Navidrome and Last.fm — the first run may take several minutes depending on library size.
  2. 2 Watch the live log output stream in real time. When the run finishes with exit code 0, results are ready.
  3. 3 Return to the Report Viewer to browse paginated cards. Use Prev / Next to navigate each section independently without reloading the page.
  4. 4 Preview an album: hover over any album art to reveal streaming icons. Click a service icon to open an embedded player inside the card. Click × to close it and return to the album art. Apple Music works with no setup; Spotify and YouTube require credentials in .env — see Streaming Previews below.
  5. 5 Each card links out to Last.fm, Discogs, Bandcamp, and YouTube Music. Use Copy to copy the artist and album title to your clipboard for searching elsewhere.
  6. 6 If SLSKD is configured, click the SLSKD button on any card to queue a search directly in your Soulseek daemon. The result appears in the SLSKD UI instantly.
  7. 7 Use the ✕ dismiss button to hide any card you've already acted on. Dismissed items are remembered across sessions and are excluded from future script runs too.

Navigating Results

Each section (Discover and Missing) has its own independent Prev / Next pager. Clicking a page button updates only that section without reloading the full page. The URL updates so you can bookmark or share a specific page, and browser Back / Forward navigate between pages as expected.

The count in the section header shows how many results are available and how many have been dismissed this session. The page indicator (e.g. 3 / 8) tells you where you are. Prev is disabled on page 1; Next is disabled on the last page.

After dismissing a card the current page reloads automatically to fill the gap. When a section has no results yet, an empty state with a link to the Run Dashboard is shown.

Scheduling

Both scripts can run automatically on a cron schedule. Set these environment variables in your .env file (5-field cron syntax):

VariableExampleMeaning
SCHEDULE_MISSING0 3 * * 0Run every Sunday at 3 AM
SCHEDULE_DISCOVER0 4 * * 0Run every Sunday at 4 AM

Leave the variables unset or blank to disable scheduling. The next scheduled run time is shown on the Run Dashboard.

SLSKD Integration

When configured, each album card shows a SLSKD button in the action bar. Clicking it queues a search for that artist and album directly in your running SLSKD instance — no copy-paste required. The search appears in the SLSKD UI for you to browse results and download.

VariableRequiredNotes
SLSKD_URL Yes Full base URL of your SLSKD instance, e.g. https://slskd.yourdomain.com or http://10.20.30.x:5030. No trailing slash.
SLSKD_API_KEY Preferred API key set in SLSKD's appsettings.yml under web.authentication.api_keys. This is simpler and more secure than username/password.
SLSKD_USER / SLSKD_PASS Fallback Used if no API key is set. The webapp obtains a JWT via POST /api/v0/session and caches it for 23 hours.

After adding credentials to Docker/music-reports/.env, restart the container — no rebuild required.

Note: A simple restart (docker compose down && docker compose up -d) picks up new environment variables without a full rebuild.

How Matching Works

Missing Popular Albums scans your local files, queries Last.fm for each artist's top albums by global playcount, and flags any album not present in your library. Results are sorted alphabetically by artist.

Discover Similar Artists uses Last.fm's artist.getSimilar API. By default (DISCOVER_SIMILARITY_MODE=lastfm), the similarity score reflects shared listener overlap — so artists from different genres can appear if their audiences overlap. Set DISCOVER_SIMILARITY_MODE=tags to re-score candidates by genre-tag Jaccard similarity instead, which eliminates cross-genre mismatches (e.g. an ambient artist appearing alongside hip-hop). DISCOVER_TAG_OVERLAP sets the minimum shared-tag threshold when using lastfm mode.

Note: In lastfm mode, scores near 100% do not guarantee a genre match — they mean Last.fm listeners who play Artist A also tend to play Artist B. Use the Discogs and Last.fm links on each card to verify before adding to your collection.

About

Cratedigger is a self-hosted tool for managing and exploring your personal music library. It runs entirely on your local network and does not send any data to third parties beyond the Last.fm and Navidrome APIs you configure.
